Concurrency & Deadlock

About

Concurrency means multiple computations are happening at the same time.

Multiple applications running on one computer.

Multiple processors in a computer (today, often multiple processor cores on a single chip)

Deadlock describes a situation where two or more threads are blocked forever, waiting for each other

Deadlock Definition: It is a situation that occurs in OS when any process enters a waiting state because another waiting process is holding the demanded resource.

Deadlock occurs when multiple threads need the same locks but obtain them in different order.

Concurrency is important because:

1.Having concurrency allows the operating system to run multiple applications at the same time.

2.Having concurrency allows the resources that are NOT being used by one application can be used for other applications.

3.Without concurrency, each application has to be run to completion before the next one can be run.

Deadlock is important because:

1.It is very important to prevent a deadlock before it can occur. So, the system checks each transaction before it is executed to make sure it does not lead to deadlock.

2.If there is even a slight chance that a transaction may lead to deadlock in the future, it is never allowed to execute.

Reason for arise of deadlock

1.Mutual Exclusion: One or more than one resource are non-shareable (Only one process can use at a time)

2.Hold and Wait: A process is holding at least one resource and waiting for resources.

3.No Preemption: A resource cannot be taken from a process unless the process releases the resource.

4.Circular Wait: A set of processes are waiting for each other in circular form.

Algorithms

Banker's Algorithm

Resource allocation and deadlock avoidance algorithm
Counting Semaphore

significant technique to manage concurrent processes
Binary Semaphore

Semaphore whose integer value range over 0 and 1

Lock Variable

Busy waiting solution which can be used for more than two processes
TSL

simply loads the value of lock variable into the local register R0 and sets it to 1 simultaneously.
Turn Variable

busy waiting solution which can be implemented only for two processes.
